# QA Standard – Severity Rules

## S1 – Critical
- Data loss
- Security vulnerability
- System crash
- Payment broken
- No workaround

## S2 – Major
- Feature broken
- Wrong calculations
- API returns incorrect data
- Workaround exists but difficult

## S3 – Minor
- UI issues
- Incorrect messages
- Layout misalignment

## S4 – Trivial
- Cosmetic
- Typos
- Low-impact UX issues
