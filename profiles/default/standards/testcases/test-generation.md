# Test Case Generation Standard

All test cases generated by the QA Agent must follow this standard to ensure consistency and importability into tools like Testmo.

## Test Case Structure

### 1. Title
- **Format**: `[Module] - [Action] - [Expected Outcome]`
- *Example*: `Login - Enter Invalid Password - Show Error Message`

### 2. Type
- Functional
- Regression
- Smoke
- API
- Security
- Performance

### 3. Priority
- **Critical**: Blocks core functionality.
- **High**: Major feature functionality.
- **Medium**: Minor feature or edge case.
- **Low**: Cosmetic or rare edge case.

### 4. Preconditions
- State required before starting the test.
- *Example*: "User is on the Login Page", "User has a valid account".

### 5. Steps
Numbered list of actions.
1. Navigate to...
2. Click on...
3. Enter...

### 6. Expected Result
Clear description of success.
- *Example*: "User is redirected to Dashboard and sees 'Welcome' message."

## Coverage Checklist
Ensure you have generated cases for:
- [ ] **Happy Path**: The standard success flow.
- [ ] **Negative Path**: Invalid inputs, error states.
- [ ] **Boundary Values**: Min/Max limits, empty states.
- [ ] **Security**: Permissions, auth checks (if applicable).
