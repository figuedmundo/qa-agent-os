Ticket Folder Structure Template
=================================

This template defines the folder hierarchy for ticket-level documentation.

Structure:
features/
  [feature-name]/
    [ticket-id]/
      documentation/
        visuals/

Details:
--------

features/[feature-name]/[ticket-id]/
  Root directory for ticket under specific feature
  Ticket ID should be normalized (e.g., WYX-123)
  Created during /plan-ticket Phase 1

documentation/
  Ticket-specific documents collected during Phase 2
  Should contain:
    - Jira ticket export or PDF
    - Ticket-specific requirements and acceptance criteria
    - Technical specifications for this ticket
    - API request/response examples
    - Other ticket-specific documentation
  Should also contain COLLECTION_LOG.md listing all collected documents

visuals/
  Subdirectory within documentation/ for visual assets
  Stores mockups, screenshots, UI designs specific to this ticket

test-plan.md
  Tactical test plan for this specific ticket
  Created during /plan-ticket Phase 3
  Contains 11 sections: References, Ticket Overview, Test Scope, Testable Requirements, Test Coverage Matrix, Test Scenarios & Cases, Test Data Requirements, Environment Setup, Execution Timeline, Entry/Exit Criteria, Revisions
  References parent feature documentation (feature-knowledge.md, feature-test-strategy.md)

test-cases.md
  Generated executable test cases from test-plan.md
  Created during /plan-ticket Phase 4 (optional) or via /generate-testcases command
  Contains detailed test case specifications with steps, expected results, test data, and execution tracking

COLLECTION_LOG.md
  Metadata document listing all documents collected for this ticket
  Created during /plan-ticket Phase 2
  Tracks: what was collected, when, from which source
